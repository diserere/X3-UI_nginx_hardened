# Лог решений по безопасности

## Архитектура маршрутизации трафика (SNI-Proxying)

### Концептуальный вызов
Технология **VLESS + Reality** принципиально несовместима с классическим реверс-проксированием на уровне L7 (HTTP), так как Xray должен контролировать TLS-рукопожатие для эмуляции стороннего сайта-маскировки (например, `yahoo.com`). Прямая терминация TLS на стороне Nginx разрушает логику Reality.

### Решение: Маршрутизация на уровне L4 (TCP)
Для одновременной работы Reality, панели управления и службы подписок на одном внешнем порту (`443`) применяется модуль Nginx `stream` с директивой `ssl_preread on`. 

Nginx считывает поле **SNI (Server Name Indication)** из пакета `Client Hello` на транспортном уровне до того, как произойдет расшифровка трафика, и перенаправляет сырой TCP-поток на соответствующий бэкенд.

#### Схема распределения потоков данных

```text
                     ┌────────────────────────────────────────┐
                     │ Входящий трафик из WAN (Порт 443, TCP) │
                     └───────────────────┬────────────────────0;
                                         │
                                         ▼
                     ┌────────────────────────────────────────┐
                     │           Nginx Stream (L4)            │
                     │            (ssl_preread on)            │
                     └───────────────────┬────────────────────0;
                                         │
                 ┌───────────────────────┴───────────────────────┐
                 │ Разделение по имени домена (SNI)              │
                 ▼                                               ▼
     [ SNI: panel.xxx.duckdns.org ]                  [ SNI: yahoo.com ]
     [ SNI: sub.xxx.duckdns.org   ]                  (или другой сайт-маскировка)
                 │                                               │
                 ▼                                               ▼
┌─────────────────────────────────┐             ┌─────────────────────────────────┐
│        Nginx HTTP (L7)          │             │          Xray Backend           │
│  (Внутренний порт, например 443)│             │     (Внутренний порт: 8443)     │
└────────────────┬────────────────┘             └─────────────────────────────────┘
                 │
         ┌───────┴───────┐
         ▼               ▼
   [Путь: /admin/]  [Путь: /sub/]
         │               │
         ▼               ▼
   ┌───────────┐   ┌───────────┐
   │ Панель    │   │ Служба    │
   │ (32180)   │   │ Подписок  │
   │           │   │ (32196)   │
   └───────────┘   └───────────┘
```

### Преимущества для безопасности
1. **Минимизация Attack Surface:** Снаружи открыт строго один порт — `443`. Порты панели и подписок полностью изолированы.
2. **Защита от сканирования по IP:** При попытке прямого обращения к IP-адресу сервера (без SNI) срабатывает правило `default`, которое отдает пустую заглушку, скрывая присутствие VPN-сервисов.
3. **Изоляция Reality:** VPN-трафик проходит через Nginx транзитом без дешифрации, что исключает появление дополнительных векторов атак на стороне веб-сервера.
